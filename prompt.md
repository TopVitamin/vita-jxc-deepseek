# 角色设定

你是一名资深前端工程师，擅长使用现代前端技术搭建企业级后台管理系统。现在需要你从0到1实现一个进销存后台管理系统前端项目。

项目名称：维他进销存

目标：搭建一个完整可运行的前端项目，包含注册、登录、后台框架、首页、基础资料、采购管理、销售管理、库存管理、系统设置等核心模块，并且每个模块都需要具备典型B端系统的增删改查能力。

---

# 一、技术栈要求

优先使用以下技术栈：

- React
- Vite
- TypeScript
- React Router
- Tailwind CSS
- shadcn/ui
- lucide-react
- Zustand 或 React Context
- TanStack Table 或自定义表格组件
- React Hook Form
- Zod

如果项目环境中无法使用shadcn/ui，可以使用原生组件+Tailwind CSS自行封装基础组件，但整体风格要保持现代、干净、克制、偏B端后台系统。

---

# 二、整体设计风格

系统定位是一个中小企业可用的进销存管理后台，不要做成花哨的大屏，也不要做成C端风格。

视觉要求：

- 整体偏中性色，黑白灰为主
- 主按钮可以使用深色或蓝色
- 状态颜色仅用于成功、失败、警告、禁用等业务状态
- 页面布局高密度、紧凑、清晰
- 保持现代、整洁、功能优先
- 表格、筛选区、表单、详情页都要符合B端后台常见交互
- 不要使用过度渐变、复杂动效、夸张阴影
- 页面要有真实业务系统的感觉，而不是Demo玩具感

# 十二、Mock数据与交互要求

由于当前不接真实后端，请使用Mock数据模拟接口。

## Mock 数据量

每个业务模块至少准备 15-25 条初始 Mock 数据，确保分页效果和筛选体验真实可用。

## Mock API 通用工厂

封装 `createCrudApi<T>` 泛型工厂函数，接收数据集和搜索配置，自动生成带以下能力的完整 CRUD 接口：

**筛选维度支持：**
* 关键词模糊搜索（可配置匹配字段列表）
* 分类筛选（category、type、region、settlement、brand 等）
* 组织筛选（org）
* 角色筛选（role）
* 字典编码筛选（dictCode）
* 业务类型筛选（bizType）
* 供应商/客户筛选（supplierName、customerName）
* 状态筛选（docStatus、status）
* 日期范围（dateFrom、dateTo，dateField 可配置，如 transferDate、operatedAt 等）
* 数值区间（minPrice/maxPrice、minStock/maxStock）

**分页与排序：** 前端模拟分页，每页默认 10 条，支持切换页码。

**网络延迟模拟：** 所有接口返回前等待 300ms，模拟真实网络环境，使 loading 状态可见。

## 交互要求

* 所有列表页面都要有初始化数据，Mock 数据在 `src/mock/data.ts` 中集中管理
* 新增后数据要能出现在列表中（插入到数据集前面）
* 编辑后列表数据要即时更新
* 删除时需要二次确认弹窗
* 启用/禁用即时切换状态，无需确认
* 查询和重置需要有基础交互，重置清空所有筛选项并重新加载全部数据
* 分页前端模拟
* 审核、反审核、关闭等操作通过更新状态实现，需弹出确认提示并说明业务影响
* 表单提交后关闭弹窗，数据实时反映在列表中
* 删除、审核等关键操作要有确认弹窗，文案需说明操作后果

Mock API 封装在 `src/mock/api.ts` 中，不要把所有业务逻辑都堆在页面组件里。
